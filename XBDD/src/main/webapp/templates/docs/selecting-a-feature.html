How to view a feature

Preparatory steps

  • In order to view a feature you must first locate the build that this feature resides in.
  • For help on how to access a build please read the help for selecting a build.

Choosing a view to suit you

1. List view

List view provides a vertical list of all the features that are tested in the current build. Clicking on a feature causes the information about the feature and a list of its scenarios to be displayed to the right of the list. Deselecting a selected feature will cause the feature and scenario information to disappear and be replaced by the build statistics.

2. Tag view

Tag view displays all of the tags that are currently attached to feature files for the particular build that you are currently viewing. Selecting a tag shows a list of all the features and scenarios that are annotated with that particular tag. The scenarios are indented with a vertical line to their left.

The status of the scenario or feature is colour coded into the view, for more information read status of scenarios and features. Note that for a scenario to appear it must be individually annotated with the tag. The feature containing that scenario is then show regardless of if the feature is annotated with that tag or not. Clicking on a feature will cause the feature to be displayed to the right of this view. Clicking on a scenario will display the feature to the right, scroll to the scenario and display it.

Deselecting a selected feature will cause the feature and scenario information to disappear and be replaced by the build statistics.

3. Folder view

Folder view displays features structured by the folders in which the feature files are stored in the project repository. Folders can also contain additional folders replicating the nested file structure. These folders can be opened or closed to display or hide the features that they contain. The status of the feature is colour coded into the view, for more information read status of scenarios and features. Clicking on a feature will cause the feature to be displayed to the right of this view. Deselecting a selected feature will cause the feature and scenario information to disappear and be replaced by the build statistics.

Filtering the results

The colourful panel of buttons above, each representing a status, allows you to filter the features that are displayed. Initially all statuses are selected and therefore all features are visible. Deselecting a filter will remove features with that status. For more information on what each of the buttons mean please read the section on status of scenarios and features.
