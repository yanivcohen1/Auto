What Status' exist and what they mean

Passed

The passed status is represented by the colour green. It symbolises that the feature, scenario or step has either passed its automated test or been manually passed.

Failed

The passed status is represented by the colour red. It symbolises that the feature, scenario or step has either failed its automated test or been manually failed.

Undefined

The undefined status is represented by the colour orange. It symbolises that the feature, scenario or step has not yet been tested. This is only possible for manual tests.

Skipped

The skipped status is represented by the colour blue. It symbolises that the feature, scenario or step has been skipped by the automated test.

Empty

The empty status is coloured grey and if it is given to a feature it means that the feature has no scenarios therefore cannot have a status. If it is given to a scenario it means the scenario has no steps and therefore cannot have a status.

Non-Existant

The non-existent status is coloured grey. It will only appear in Merge View and means that the feature or scenario shown does not exist in the build that the symbol is aligned with.

How feature statuses are calculated

  • Failed: If a scenario with the feature has a failed statues
  • Passed: If all scenarios within a feature that are not empty are passed
  • Undefined: If no scenarios within the feature have a failed status and a scenario within the feature has an undefined status
  • Skipped: If no scenarios within the feature have a failed or undefined status and a scenario within a feature has a skipped status
  • Empty: If all scenarios within a feature have an empty status
