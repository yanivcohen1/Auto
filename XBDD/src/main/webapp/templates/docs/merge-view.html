Viewing the combined status of multiple builds

Preparatory steps

In order to view multiple builds you must open up the build navigation. For details on how to do this please read the Viewing Builds section.

The Selection Process

To enter the multiple build section view, press the build button in the top left of the build navigator. The builds will all fade out until they are selected. The selected builds are also emphasised with a green tick to the right of the selected build. Initially the blue "Merge View" button at the bottom will be disabled until a minimum of two builds have been selected. Once the builds that you want to view the merged status of have all been selected press the "Merge View" button.

Loading the View

The preparation and rendering of this view can take some time depending on the number of builds selected and the number of scenarios and features present in each build. You will be presented with a message saying "loading, please wait". If an error does occur attempting to merge the builds the message will change to notify you.

What this view means

The ordering of builds is based on the current ordering of builds for that product. For more details on build ordering refer to the reordering builds page. Only features and scenarios that are present in the most recent build selected are displayed in this view even if they are present in a previous build. The name of the features and scenarios are displayed in the left most column of the merge build table. A row representing a feature is highlighted with the colour of the merged status of the feature. Scenarios are sightly indented. The merged statuses are displayed as the second column on the left, with larger icons and text than the other builds.

The merge status of a scenario is calculated by looking back from the most recent build at the status of the scenario in previous builds. The fist occurrence of a passed, failed or empty scenario is given the merge status scenario. If none of these statuses are present then the merged scenario receives the status of the most recent build. The status of a merged feature is calculated by in the normal way by reducing the status of the features scenarios. For more information on how feature status is calculated please refer to the section to status of scenario and features. For information on what each of the icons mean also refer to status of scenario and features.

The "Show All" and "Hide All" buttons located on the top left of the page can be used to show or hide all of the scenarios leaving only the features. The feature names are clickable to collapse and expand the view of the scenarios that belong to that feature.

Printing the merge view

You can print this page using

File -> Print

Or the relevant keyboard shortcut (Cmd + p or Ctrl + p)

We suggest for a better result that when printing this page set the page orientation to landscape.

If there are a large number of builds in the merge they may not all show up when printing however the merge result and most recent builds should appear.
