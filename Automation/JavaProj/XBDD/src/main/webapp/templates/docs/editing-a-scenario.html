How to edit a scenario's testing information

Preparatory steps

In order to edit a scenario you must first locate the build and feature that this scenario presides in.

  • For help on how to access a build please read the help for selecting a build
  • For help on how to access a feature please read the help for selecting a feature
  • For help on how to access a scenario please read the help for selecting a scenario

Testing tips

Testing tips are used by testers to provide guidance on how to carry out a particular step in a test. Testing tips are located in the information presented when a scenario is selected and are specific to that scenario. For more information see selecting a feature. The testing tips text area can be resized and also allows for the addition of markdown for more descriptive testing tips. For documentation on the markdown click here. You can toggle into view and edit mode by selecting the blue eye and pen respectively. Testing tips are carried forward between builds.

Environment

The environment information is presented when a scenario is selected and is used by testers to explain what environment the test was carried out in. Commonly this may just be a link to the documentation for a particular virtual machine that the tests will be run on. For more information see selecting a feature. You can toggle into view and edit mode by selecting the blue eye and pen respectively.

The environment information is not carried forward though builds but is specific to the build that is edited. Each environment that is input is stored and when typing an auto complete drop down will appear making suggestions of environments that have been used in other builds for this product which can be selected. The drop down displays suggestions based on previous environments that contain the same string as currently input.

Execution Notes

The execution notes are presented when a scenario is selected. The execution notes are used by testers to give details of occurrences during a manual test. For more information see selecting a feature. You can toggle into view and edit mode by selecting the blue eye and pen respectively. The execution notes are not carried forward though builds but is specific to the build that is edited.

Attachments

Attachments are used to provide additional information about testing a given scenario. An attachment can be uploaded by clicking the blue paperclip icon. This is located within the information about a given scenario. For more information see selecting a feature. Attachments are not carried forward to future builds but instead they are particular to the given build.

Steps

The steps of a scenario are visible when a scenario is selected. For details on changing the status of a scenario please read the section on manual testing.

Saving changes

Once you have edited this field you must click the save button in the blue container at the top of the screen for the changes to take effect. You can discard changes by clicking the discard button located in the save blue container.
