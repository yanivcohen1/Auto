Attaching a Protractor test to an existing selenium session

Let’s say that only a portion of your website uses Angular and there are already existing selenium tests. You would like to use Protractor testing because you’ve heard great things about Protractor testing for Angular. However, after a few prototypes you notice that the browser session is created at the beginning of the test and destroys the browser session at the end of the test. This doesn’t fit your app’s workflow.

Instead of reinventing all your tests in Protractor, you can hand off the browser session to run a portion of your tests in Protractor.

Here’s what you’ll need:

  • Look up the selenium session id for the launched browser
  • Pass the selenium session id to Protractor via the configuration file or via the command line interface

While I was writing the tests for this feature, I found that navigating to localhost:4444/wd/hub to find / create a selenium session. Also, selenium server has an api to get existing selenium sessions.

In this example we’ll run through the sample attach session test in the Protractor repository. Using the command line interface, you could attach the selenium session with:

protractor spec/attachSession.js --seleniumSessionId=df573f9d-843e-47d0-a56b-00f54ab384f3

Or you could edit the configuration file:

exports.config = {
  seleniumAddress: 'http://localhost:4444/wd/hub',
  seleniumSessionId: 'df573f9d-843e-47d0-a56b-00f54ab384f3',     
  framework: 'jasmine',
  specs: [
    'attachSession_spec.js'
  ],
  capabilities: {
    browserName: 'chrome'
  }
}

After automating this test, I have found passing the variable via command line easiest. These files are located:

protractor/
|- scripts/
|  |- attachSession.js
|- spec/
|  |- attachSessionProvider/
|  |  |- attachSession_spec.js
|  |- attachSession.js
|- testapp/
|- package.json
  • scripts/attachSession.js is a node script that sets up the selenium session, runs the protractor test and tears down the selenium session.
  • spec/attachSession.js is the configuration file for the protractor test
  • spec/attachSessionProvider/attachSession_spec.js is a simple Angular 2 test app
  • testapp/ is the Angular 1 and 2 app used for testing
  • package.json has a script to start the test app

To run this test:

  • make sure that the selenium webdriver is started with webdriver-manager start
  • start the test app with npm start
  • run the node script with node scripts/attachSession.js

Attach Session details

Digging into more detail what’s going on in Protractor. The launcher calls the initFn function and creates a new config parser. The config parser resolves and returns the configuration from the command line interface and Protractor configuration file. The launcher then calls the runner.

The runner constructor calls the loadDriverProvider_ method and passes around the configuration. The loadDriverProvider_ method checks to see if the selenium address and the selenium session id is set. If these two items are present, the attach session provider gets the webdriver using the attachToSession method. When the driver quits, the method does not quit the existing browser session.

Alternatively, if this was a hosted driver, the webdriver createSession method creates a brand new browser. Also, when it calls quitDriver it calls the default method in the driverProvider and closes the browser session.
